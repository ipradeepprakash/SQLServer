Basic Diagnostics: 
Check Hardware information
https://www.sqlskills.com/blogs/glenn/sql-server-diagnostic-information-queries-detailed-day-8/

Check SQL instance related details
		○ Server name, 
		○ instance name, 
		○ edition, 
		○ product version, 
		○ product level,
		○ clustered or not, 
	
	Ex: 
		Select serverproperty ('ComputernamePhysicalNetBios') as 'ServerName'
		Select serverproperty ('edition') as ''
		Select serverproperty ('productlevel') as 'product level'
		Select serverproperty ('productversion') as 'product Build version'
		Select serverproperty ('IsClustered') as 'Clustered'
		Select serverproperty ('instancename') as 'instance name'
	
Check Consistency / integrity: DBCC Commands (database console commands)
		○ DBCC CHECKDB: checks the integrity and consistency checks in all data pages for any corruption.  
	                      > Best to execute before backups.
		○ DBCC CHECKTABLE
		○ DBCC SQLPERF(logspace)

Check active connections:
		• Legacy commands:
			○ Sp_who, sp_who2 active
			○ Sys.sysprocessess etc
		• DMVs
			○ Sys.dm_exec_connections
			○ Sys.dm_exec_requests
			○ Sys.dm_os_sys_info
			○ Sys.dm_os_wait_stats
			○ Sys.dm_io_virtual_file_stats …  etc
		• Event viewer
			○ Info about OS level errors, application level errors.
		• SQL Errorlog
			○ Info on database level errors, I.e. login failures, filegroup full, startup/shutdown, transaction log full, deadlocks
		• Any changes on - SQL Server properties
			○ Sp_configure : to check SQL server Max Memory, audit settings, advanced parameters etc.
		• Blocked processes / deadlocks
		• Locks
